##1. サービス概要

[介護のレク365]は、高齢者向けのAIレクリエーション支援サービスです。

介護施設や在宅介護で、AIキャラクターが口腔体操・軽い体操・「今日は何の日」の小話を進行し、高齢者の楽しみづくりと介護者の業務負担軽減を支援します。

##2. このアイデアはどこから生まれたか
###2-1. きっかけとなった体験・感情

私は介護現場で働いていた経験があります。

施設では、利用者の方が日中にテレビを視聴して過ごす時間がとても長くありました。テレビは利用者の方が集中して見てくれるため、転倒などの事故を防ぎやすく、介護職員もその間に他の利用者の介助や見守りを行うことができます。

一方で、リハビリやレクリエーションは毎日十分に実施できるわけではありません。また、病気の後遺症による麻痺などで以前のように趣味や日常生活を楽しめなくなり、「何もする気が起きない」と話す利用者の方も多く、結果としてテレビを見る時間が長くなっていました。

しかし、テレビでは戦争や事故、事件などの暗いニュースが多く放送されます。ある日、リハビリ職員から「暗いニュースを見続けることは利用者に悪影響を与えるのではないか」という意見が会議で挙がりました。

実際に、ニュースをきっかけに過去のつらい記憶を思い出して涙を流したり、不安になったり、怒り出してしまう利用者の方を何度も見てきました。

そこで私は、テレビと同じように自然と見続けられ、暗いニュースではなく、高齢者が安心して楽しめるコンテンツを提供できるサービスがあれば、利用者にも介護職員にも役立つのではないかと考え、このサービスを思いつきました。

###2-2. なぜそれが「気になった」のか

私が勤めていた施設では、お看取りを行っていました。

どれだけ元気だった利用者の方でも、いつかは最期の時を迎えます。その姿を何度も見てきたからこそ、一日一日を大切に過ごしてほしいという思いが強くなりました。

そのため私は、利用者の方に「今日も楽しかった」と思って一日を終えてもらいたいと考えています。

しかし、介護職員は利用者のために時間を使いたいと思っていても、食事介助や排泄介助などの業務に追われ、十分にレクリエーションを行えない場面を何度も見てきました。

だからこそ、介護者の負担を減らしながら、高齢者が毎日楽しめる仕組みがあれば、利用者にとっても介護職員にとっても価値のあるサービスになると考えました。

##3. 課題の整理
###3-1. 表に見えている困りごと
・レクリエーションを毎日企画・実施することが難しい
・利用者がテレビを見るだけで一日を過ごすことが多い
・暗いニュースによって、不安や悲しい記憶が呼び起こされる利用者がいる

###3-2. 本当に解決したい課題は何か

-なぜ利用者はテレビを見る時間が長くなるのか？

→ レクリエーションを毎日実施することが難しく、楽しめる活動が限られているから。

-なぜレクリエーションを毎日実施できないのか？

→ 介護職員は食事や排泄などの介助業務に追われ、十分な時間を確保できないから。

-なぜそれが問題なのか？

→ 利用者は一日を受け身で過ごす時間が増え、心身を動かす機会や「今日も楽しかった」と感じられる時間が減ってしまう。また、介護職員も本来提供したいケアを十分に実践できない。

--本当に解決したい課題

介護職員が忙しい環境でも、高齢者が毎日安心して楽しめる時間を過ごし、「今日も楽しかった」と思える環境をつくること。

##4. 想定ユーザーについて
###4-1. 想定しているユーザー
-年齢

25〜40 歳

-理由

デイサービスや介護施設でレクリエーションや食事介助を担当することが多い年代であり、業務効率化のためのICTツールにも比較的抵抗なく取り入れやすいためです。

-生活・状況

デイサービスで働く介護職員。食事前は口腔体操を実施したいと思っているが、排泄介助や食事介助の準備などが重なり、十分な時間を確保できない。また、毎日のレクリエーション内容を考えることにも負担を感じている。

-理由

アプリを起動するだけで、その日の口腔体操やレクリエーションをAIが進行するため、介護職員は利用者への介助や準備を進めながらレクリエーションを実施できます。

-環境

デイサービスや介護施設の共有スペースで、テレビやタブレットを利用して複数人の利用者へレクリエーションを提供している。

-理由

大きな文字やシンプルな画面設計にすることで、テレビへ映しても見やすく、職員もワンタップで利用できるようにします。

###4-2. 自分とユーザーの距離

過去の自分

私は介護職員として働いていた経験があり、食事前の忙しさやレクリエーションを毎日考える大変さを実際に経験しました。そのため、現場で働く介護職員が本当に必要としている機能や使いやすさを意識してサービスを設計しています。

###4-3. 実際に使われる可能性

最初に利用してくれるのは介護施設やデイサービスの介護職員を想定しています。

食事前やレクリエーションの時間になると、「今日は何をやろう」と考える代わりに、このアプリを起動することを想定しています。

現在はYouTubeの体操動画やテレビ番組を利用している施設もありますが、高齢者向けに特化した内容ではないことや、毎日同じ動画になりやすいという課題があります。

本サービスでは、介護現場に特化した内容をワンタップで利用できるため、準備時間の短縮とレクリエーションの質の向上を両立できると考えています。

###4-4. 導入・継続の流れ

介護施設では、「毎日のレクリエーションを考える時間がない」と感じたことが導入のきっかけになります。

職員はテレビやタブレットでアプリを起動し、「今日のレク」を選択するだけで、AIキャラクターが体操や「今日は何の日」の小話を進行します。その間、職員は食事の準備や利用者の介助を進めることができます。

毎日異なるレクリエーションや季節に合わせた話題が配信されるため、利用者は「今日はどんな内容だろう」と楽しみにしながら継続して利用できます。

個人開発サービスであることから、まずはポートフォリオとして介護現場の知人や元同僚に利用してもらい、実際の意見を取り入れながら改善を重ねていくことを想定しています。

###5-2. それでも自分が作りたい理由

YouTubeには多くの動画がありますが、介護施設で毎日利用するには動画を探す手間があり、広告が表示されることもあります。また、「今日は何の日」や口腔体操などを一つのサービスでまとめて提供しているわけではありません。

私は介護現場で実際に働いた経験から、「忙しい介護職員でも、ワンタップでレクリエーションを始められるサービス」を作りたいと考えました。

###5-3. 差別化

このアプリは、食事前の忙しい時間帯にレクリエーションの準備まで手が回らない介護職員が、ワンタップで高齢者へ安心して楽しい時間を提供できることを価値とするサービスです。

##6. このサービスで提供したい価値
###6-1. ユーザーの変化
-行動の変化

介護職員はレクリエーションの準備時間を減らしながら、毎日継続して口腔体操や体操を実施できるようになります。

-気持ちの変化

利用者は毎日違うレクリエーションを楽しみにし、「今日も楽しかった」と感じられる時間が増えます。介護職員も「やりたかったケアができた」という達成感を得られます。

-考え方の変化

レクリエーションは負担ではなく、高齢者との大切な時間として前向きに取り組めるようになります。

###6-2. 価値を一文で表す

このサービスは、レクリエーションの準備に十分な時間を確保できない介護職員が、ワンタップで高齢者へ毎日楽しい時間を届けられるようになるサービスです。

###10-2. 技術選定
候補	採用	理由
Ruby on Rails	採用	学習済みで開発スピードが速く、ポートフォリオとしても実装しやすいため
React	不採用（本リリース後検討）	キャッチアップに時間がかかるため
Vue	不採用	今回はRails中心で完成度を優先したいため
今回チャレンジしたいこと
AIを活用したレクリエーションの提供
高齢者向けUI/UX設計
ImageMagickによる画像加工
動画コンテンツ管理
不安な点

AI動画生成やキャラクターの実装は難易度が高いため、まずはMVPで体操動画や字幕表示を実装し、その後段階的にAI機能を追加していく予定です。

###10-3. キャッチアップ不足の懸念

AI機能や動画生成技術は未経験の部分が多く、実装には学習時間が必要です。

そのため、MVPではRailsを中心に基本機能を完成させ、本リリースでAIや音声機能を段階的に追加する計画としています。
